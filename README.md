# Developer Portfolio API (Go + Fiber v3)

Бэкенд-сервис для лендинг-презентации разработчика с полноценным REST API, файловой аналитикой, защитой от спама (Rate Limiting) и интеграцией с искусственным интеллектом (OpenAI GPT).

Проект разработан в рамках тестового задания с фокусом на высокую производительность, чистоту архитектуры и надежность (Graceful Fallback).

## 1. Как запустить проект

### Требования

* Go 1.21 или выше
* Аккаунт OpenAI (и API ключ) или доступ к прокси-серверу API
* Настройки SMTP (для отправки email-уведомлений)

### Инструкция по установке и запуску локально

1. **Клонируйте репозиторий:**

   ```bash
   https://github.com/vitaly06/portfolio-rest-api
   cd portfolio-api

   ```
2. **Настройте переменные окружения:**
   Создайте файл `.env` в корневой директории проекта на основе примера:

   ```env
   PORT=8080
   OPENAI_API_KEY=your_openai_api_key_here
   SMTP_HOST=smtp.mail.ru
   SMTP_PORT=587
   SMTP_USER=your_email@mail.ru
   SMTP_PASSWORD=your_smtp_app_password_here
   OWNER_EMAIL=your_email@mail.ru


   ```
3. **Установите зависимости:**

   ```bash
   go mod tidy

   ```
4. **Запустите сервер:**

   ```bash
   go run cmd/api/main.go
   ```

   Сервер запустится по адресу: http://localhost:8080

## 2. Стек технологий

* **Язык разработки:** Go (Golang) — выбран за его высокую производительность, эффективную работу с многопоточностью (горутины) и низкое потребление памяти под нагрузкой.
* **Веб-фреймворк:** Fiber v3 — один из самых быстрых и современных веб-фреймворков для Go, построенный на базе fasthttp, обеспечивающий экстремально низкий аллокационный след.
* **Валидация:** github.com/go-playground/validator/v10 — индустриальный стандарт для безопасной проверки входящих структур.
* **Конфигурация:** github.com/joho/godotenv — для прозрачной работы с .env файлами.
* **AI Инструменты:** OpenAI Chat Completions API (модель gpt-3.5-turbo).

## 3. Архитектура проекта

Проект спроектирован по принципам слоистой чистой архитектуры (Layered/Clean Architecture). Это гарантирует слабую связанность компонентов и изоляцию бизнес-логики:

1. **cmd/main.go** — Точка входа. Отвечает за инициализацию конфигурации, сборку слоев (Dependency Injection) и запуск HTTP-сервера.
2. **Delivery (HTTP Handlers / Middleware)** — Слой представления. Принимает HTTP-запросы, отвечает за роутинг, сериализацию JSON, CORS, логирование и Rate Limiting.
3. **Usecase (Services)** — Слой бизнес-логики. Здесь сосредоточена ключевая координация приложения: вызов AI-анализатора, запуск асинхронной отправки почты и сохранение метрик.
4. **Repository (Data Access)** — Слой работы с данными. На данный момент инкапсулирует работу с файловой системой (чтение/запись JSON и логов). Абстрагирован интерфейсным подходом — при необходимости заменить хранилище на PostgreSQL, код бизнес-логики останется неизменным.

### Паттерны проектирования:

* **Dependency Injection (Внедрение зависимостей):** Все зависимости (конфиг, репозитории, сервисы) передаются в конструкторы явно, что упрощает юнит-тестирование.
* **Async Workers (Асинхронные воркеры):** Тяжелые операции (отправка писем через SMTP) вынесены в отдельные горутины (go func()), благодаря чему клиент получает HTTP-ответ мгновенно, не дожидаясь сетевых таймаутов почтовых серверов.

## 5. AI-интеграция и Fallback

Бэкенд выполняет комплексную AI-задачу: анализ тональности комментария (Sentiment Analysis) + динамическая генерация персонализированного автоответа.

### Промпт (System/User):

> "Ты — AI-ассистент. Твоя задача — проанализировать комментарий пользователя и сгенерировать автоответ. Определи тональность комментария (доступно только 3 варианта: 'positive', 'neutral', 'negative'). Напиши вежливый, профессиональный краткий ответ от лица Виталия. Верни ответ СТРОГО в формате JSON без markdown-разметки, содержащий поля 'sentiment' и 'reply'. Комментарий: \"[Текст_комментария]\""

### Механизм Graceful Fallback

Если у OpenAI API превышены лимиты (Rate Limit), отсутствует интернет-соединение, либо токен заблокирован, приложение не падает. Срабатывает внутренний перехватчик ошибок:

1. Ошибка логируется в файл для администратора.
2. Система мгновенно подменяет ответ статическим шаблоном заглушки:
   `sentiment: "neutral"`
   `reply: "Здравствуйте! Спасибо за ваше обращение. Я получил ваше сообщение и свяжусь с вами в ближайшее время..."`
3. Пользователь успешно получает красивое письмо и ответ на фронтенде, стабильность бизнес-логики сохраняется на 100%.

## 6. Что сделано с помощью AI

При разработке данного сервиса ИИ использовался как инструмент повышения продуктивности (Copilot-сценарий):

* **Генерация структур данных и бойлерплейта:** Базовые скелеты для JSON-структур были набросаны нейросетью.
* **Что исправлялось вручную:**
  * Нейросети часто пытаются вернуть JSON внутри Markdown блоков (\`\`\`json ... \`\`\`), что приводило к ошибкам парсинга в Go. В промпт было добавлено жесткое ограничение, а парсер был переписан на строгий `json.Unmarshal`.
  * Переработка контекстов: стандартные библиотеки работы с OpenAI подтягивали тяжелые зависимости, поэтому интеграция была переписана на чистый, нативный `http.Client` для экономии ресурсов бинарного файла Go.

## 7. Хранение данных, Логирование и Безопасность

* **Логирование запросов:** Реализовано через кастомную Middleware в `internal/delivery/http/middleware.go`. Каждый входящий запрос логируется в файл `data/app.log` в формате JSON (`Timestamp`, `Method`, `Path`, `IP`, `Status`). Запись происходит асинхронно.
* **Rate Limiting (Защита от спама):** Реализован на уровне роутера Fiber с помощью скользящего окна. Для эндпоинта `/api/contact` выставлено ограничение: максимум 3 запроса в минуту с одного IP. При превышении лимита отдается `429 Too Many Requests` без вызова тяжелой логики отправки почты и AI, что защищает баланс OpenAI и почтовый ящик от заспамливания.
* **Статистика:** Все данные эндпоинта `/api/metrics` инкрементируются при каждом запросе в потокобезопасном режиме (благодаря `sync.Mutex`) и сохраняются в файл `data/stats.json`.
